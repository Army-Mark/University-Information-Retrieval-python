# 学校信息管理系统功能文档

## 1. 系统概述

本系统是一个功能完整的学校信息管理系统，提供学校信息的搜索、展示、管理以及用户账户管理等功能。系统采用Flask框架开发，支持Docker容器化部署。

## 2. 核心功能模块

### 2.1 学校信息搜索与展示

**功能描述**：
- 实时搜索建议：输入时自动显示匹配结果
- 多维度匹配：支持按学校ID和名称搜索
- 智能排序：精确ID匹配 > 部分ID匹配 > 名称匹配
- 搜索结果展示：显示学校ID、名称及匹配类型标记

**技术实现**：
- 后端：Flask路由 `/search` 处理搜索请求
- 前端：AJAX实时请求，动态显示搜索建议
- 匹配算法：多维度匹配逻辑，智能排序

**特色亮点**：
- 实时搜索建议提升用户体验
- 多维度匹配确保搜索准确性
- 智能排序提高搜索效率

### 2.2 学校详情查看

**功能描述**：
- 详细信息展示：学校基本信息、地址、类别、性质等
- 排名数据可视化：软科综合、校友会综合、QS世界等排名
- 标签系统：显示学校特色标签（如985、211等）
- Logo展示：显示学校Logo图片

**技术实现**：
- 后端：Flask路由 `/university/<id>` 处理详情请求
- 前端：模板渲染，展示详细信息
- 数据处理：排名数据格式化，标签提取

### 2.3 用户认证系统

**功能描述**：
- 登录功能：用户名密码登录
- 登出功能：安全退出
- 登录状态检查：页面加载时自动检查
- 会话管理：使用Flask session管理登录状态

**技术实现**：
- 后端：Flask路由 `/login`、`/logout`、`/check_login`
- 前端：登录模态框，AJAX请求
- 安全措施：会话管理，密码验证

### 2.4 账户管理（管理员功能）

**功能描述**：
- 账户列表查看：查看所有用户账户
- 账户添加：添加新用户
- 账户更新：修改现有用户信息
- 账户删除：删除用户账户

**技术实现**：
- 后端：Flask路由 `/get_accounts`、`/add_account`、`/update_account`、`/delete_account`
- 前端：账户管理界面，AJAX请求
- 权限控制：登录状态检查

**特色亮点**：
- 完整的用户管理功能
- 支持管理员对账户的全面控制
- 实时反馈操作结果

### 2.5 学校信息管理（管理员功能）

**功能描述**：
- 学校信息编辑：修改学校详细信息
- 学校信息添加：添加新学校信息
- 学校信息删除：删除现有学校
- Logo上传：为学校上传Logo图片

**技术实现**：
- 后端：Flask路由 `/edit/<id>`、`/save`、`/add_school`、`/delete_school`、`/upload_logo`
- 前端：编辑界面，表单提交，文件上传
- 数据处理：数据库操作，文件处理

**特色亮点**：
- 支持Logo上传功能
- 完整的学校信息CRUD操作
- 实时数据验证

### 2.6 滚动数据展示

**功能描述**：
- 实时滚动：学校信息滚动展示
- 滚动位置保存：记录和恢复滚动位置
- 数据加载：按需加载学校数据

**技术实现**：
- 后端：Flask路由 `/api/scrolling_data`、`/api/scrolling_position`
- 前端：JavaScript滚动效果，AJAX请求
- 数据处理：滚动位置计算，数据缓存

**特色亮点**：
- 滚动位置记忆功能，提升用户体验
- 实时数据更新
- 流畅的滚动效果

### 2.7 系统配置与部署

**功能描述**：
- Docker支持：提供Dockerfile，支持容器化部署
- 环境变量配置：通过环境变量配置数据库路径、端口等
- 响应式设计：适配不同屏幕尺寸

**技术实现**：
- Dockerfile：容器化配置
- 环境变量：数据库路径、端口、主机等配置
- 前端：响应式CSS设计

**特色亮点**：
- 完整的Docker部署支持
- 灵活的环境配置
- 跨设备兼容

## 3. 技术架构

### 3.1 后端技术
- **框架**：Flask
- **数据库**：SQLite
- **缓存**：内存缓存
- **部署**：Docker

### 3.2 前端技术
- **基础**：HTML5 + CSS3 + JavaScript
- **交互**：AJAX
- **响应式**：媒体查询
- **UI**：自定义样式

## 4. 系统流程

### 4.1 搜索流程
1. 用户在搜索框输入关键词
2. 前端发送AJAX请求到 `/search` 接口
3. 后端根据关键词匹配学校数据
4. 前端接收搜索结果并显示建议列表
5. 用户点击建议项或执行搜索
6. 系统跳转到对应的学校详情页

### 4.2 登录流程
1. 用户点击登录按钮
2. 前端显示登录模态框
3. 用户输入用户名和密码
4. 前端发送AJAX请求到 `/login` 接口
5. 后端验证用户信息
6. 前端根据响应更新登录状态

### 4.3 学校管理流程
1. 管理员登录系统
2. 进入账户管理或学校管理界面
3. 执行添加、编辑、删除操作
4. 前端发送AJAX请求到对应接口
5. 后端处理请求并更新数据库
6. 前端显示操作结果
